环境

环境支持帮助 FuelPHP 以及你的应用程序基于环境做出不同的选择. FuelPHP 通过当前激活环境设置加载/覆盖额外的配置信息.

环境列表

FuelPHP 有四个预定义环境.

  • \Fuel::DEVELOPMENT
    开发环境.
  • \Fuel::TEST
    测试环境.
  • \Fuel::STAGE
    临时环境.
  • \Fuel::PRODUCTION
    生产环境.

设置你的环境

通过设置 FUEL_ENV 服务器变量或者手动设置 fuel/app/bootstrap.php 文件来设置环境. 

// Inside app/bootstrap.php

/**
 * Your environment.  Can be set to any of the following:
 *
 * Fuel::DEVELOPMENT
 * Fuel::TEST
 * Fuel::STAGE
 * Fuel::PRODUCTION
 */
Fuel::$env = (isset($_SERVER['FUEL_ENV']) ? $_SERVER['FUEL_ENV'] : Fuel::DEVELOPMENT);

环境和配置

基于环境应用程序设置为，配置类使用环境定义的配置文件，这个配置文件将使用和环境名相同命名目录中查找。这是一个例子说明: 

app/
  config/
    stuff.php
      development/
        stuff.php
      production/
        stuff.php

当环境设置为 \Fuel::DEVELOPMENT, 从 stuff.php 设置项将会从 development/stuff.php 读取. 如果环境设置 \Fuel::DEVELOPMENT 被设置为任何其他, 其余的配置文件不会被加载. 这同样试用于其他任何环境设置.

一个此示例数据库配置真实案例. 没有默认配置选项 (有危险). 仅有环境定义配置设置.

使用 Apache 配置

你可以使用 Apache 配置语句 SetEnv 为你应用程序设置环境. 在大多数情况下来说你可能使用环境变量会容易些, 而在你没有访问服务器命令行程序权限时，这是唯一选择. 

// run this application in production mode
SetEnv FUEL_ENV PRODUCTION

环境变量名称 FUEL_ENV 以及环境定义值都需要使用大写.
